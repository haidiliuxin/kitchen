# 厨房助手 MVP 验收清单

## 1. 文档目的
这份文档用于当前项目的 MVP 阶段验收，包含两部分：

- `MVP 验收清单`
- `当前已实现功能说明`

当前定位：

- 本项目已经可以作为一个简单的 `MVP` 进行阶段性验收
- 重点验证的是 `主流程是否跑通`、`页面职责是否清楚`、`核心闭环是否成立`
- 当前仍有部分数据和接口为占位实现，属于 `可演示、可验收、可继续扩展` 的阶段

---

## 2. MVP 验收范围
本轮 MVP 主要覆盖以下模块：

- 首页
- 菜谱详情页
- 做菜步骤页
- 完成页
- 搜索页
- 导入页
- 社区页
- 我的页

---

## 3. MVP 验收清单

### 3.1 首页与导航
- [ ] 能正常打开首页
- [ ] 底部导航可切换 `首页 / 交流 / 我的`
- [ ] 首页顶部搜索入口可进入搜索页
- [ ] 首页导入入口可进入导入页
- [ ] 首页热门菜品轮播可正常滑动和切换
- [ ] 首页菜谱卡片点击后可进入菜谱详情页
- [ ] 首页分类下拉可正常展开、收起、选择
- [ ] 首页“专题推荐 / 最近做过 / 更多筛选”入口可正常进入对应内容
- [ ] 若存在未完成做菜记录，首页可显示“继续上次做菜”

### 3.2 菜谱详情与做菜流程
- [ ] 从首页或搜索结果进入菜谱详情页后，页面从顶部开始显示
- [ ] 菜谱详情页可查看：
  - [ ] 基本信息
  - [ ] 食材
  - [ ] 工具
  - [ ] 详细步骤
  - [ ] 易错点 / 补救提示
- [ ] 菜谱详情页可点击“进入跟做模式”
- [ ] 菜谱详情页可点击收藏
- [ ] 做菜步骤页可正常切换步骤
- [ ] 做菜步骤页计时器可开始 / 暂停 / 重置
- [ ] 做菜步骤页语音控制按钮可切换状态
- [ ] 做菜步骤页 AI 教练区域可正常显示
- [ ] 做菜完成后能进入完成页

### 3.3 完成页闭环
- [ ] 完成页能显示当前菜完成结果
- [ ] 完成页能显示累计完成次数
- [ ] 完成页可执行：
  - [ ] 再做一次这道菜
  - [ ] 返回菜谱详情
  - [ ] 查看最近做过
  - [ ] 去交流区写心得
  - [ ] 收藏这道菜
- [ ] 完成页推荐菜谱可点击进入对应菜谱详情

### 3.4 搜索闭环
- [ ] 首页点击搜索入口可进入搜索页
- [ ] 搜索页可输入关键词
- [ ] 热搜词点击后可直接触发搜索
- [ ] 分类按钮点击后可触发搜索
- [ ] 搜索结果区可根据关键词显示匹配菜谱
- [ ] 搜索结果为空时有空状态提示
- [ ] 搜索结果点击后可进入菜谱详情页
- [ ] 从菜谱详情返回后可回到原搜索上下文
- [ ] 搜索页可显示最近搜索
- [ ] 最近搜索可点击重新搜索
- [ ] 最近搜索可一键清空

### 3.5 导入闭环
- [ ] 首页导入入口可进入导入页
- [ ] 导入页支持切换 `视频链接 / 文档链接`
- [ ] 输入链接后可点击“开始识别”
- [ ] 开始识别后，页面中央出现识别状态弹窗
- [ ] 识别中背景会变暗
- [ ] 识别进度到 100% 后识别弹窗自动隐藏
- [ ] 识别成功后可显示结果列表
- [ ] 识别结果可查看预览
- [ ] 识别结果可进入现有菜谱详情页
- [ ] 输入非法链接时可弹出失败弹窗
- [ ] 失败弹窗包含“重新识别 / 先返回修改”
- [ ] 导入历史页可正常打开
- [ ] 导入历史中的已生成结果可再次打开

### 3.6 社区闭环
- [ ] 底部“交流”页已升级为社区页
- [ ] 社区首页可查看帖子流
- [ ] 社区首页可进入发心得页
- [ ] 社区首页可查看热门话题
- [ ] 帖子可点击进入详情页
- [ ] 帖子详情页可点赞
- [ ] 已点赞状态可保留并显示“已点赞”
- [ ] 帖子详情页可进入“我也写一条”
- [ ] 发布页可填写标题、标签、正文
- [ ] 发布页可正常发布内容
- [ ] 发布后帖子会进入社区流

### 3.7 我的页面
- [ ] “我的”页可正常显示登录 / 退出区域
- [ ] “我的”页可显示“我的收藏”
- [ ] “我的”页可显示“最近做过”
- [ ] “我的”页可显示“导入历史入口”
- [ ] “我的”页可显示“我的发布”
- [ ] 点击“我的收藏”中的菜谱可进入详情页
- [ ] 点击“最近做过”中的菜谱可进入详情页
- [ ] 点击“导入历史入口”可进入导入历史页
- [ ] 点击“我的发布”中的帖子可进入社区帖子详情

## 4. 当前已实现功能说明

### 4.1 首页
已实现：

- 首页推荐内容展示
- 热门菜品轮播
- 菜谱库列表
- 分类下拉
- 专题入口
- 最近做过入口
- 更多筛选入口
- 顶部搜索入口
- 导入入口
- 继续上次做菜入口

说明：

- 首页目前承担的是 `导航中心 + 推荐入口页`
- 不再把所有内容都直接堆在一个长页面里

### 4.2 菜谱详情页
已实现：

- 菜谱头图和基础信息
- 食材列表
- 工具列表
- 详细步骤预览
- 易错点 / 补救提示
- 进入跟做模式按钮
- 收藏按钮

说明：

- 当前详情页已经能承接首页、搜索页、导入结果页的进入流量

### 4.3 做菜步骤页
已实现：

- 当前步骤展示
- 步骤切换
- 朗读步骤
- 计时器
- 语音控制按钮
- AI 教练区域
- 完成做菜入口

说明：

- 做菜流程已经能独立跑通，是当前产品主链路的核心页面

### 4.4 完成页
已实现：

- 完成结果展示
- 累计完成次数

- 再做一次

- 查看最近做过
- 去社区写心得
- 收藏这道菜
- 下一道推荐

说明：

- 完成页现在不是单纯结束页，而是 `完成后的分流页`

### 4.5 搜索页
已实现：

- 搜索输入
- 热搜榜单
- 推荐分类
- 搜索结果列表

- 最近搜索

- 当前排序是轻量规则排序：
  - 标题强匹配优先

### 4.6 导入页

- 导入接口调用占位
- 识别状态弹窗

- 结果预览页
- 进入菜谱详情页

说明：

### 4.7 社区页
已实现：

- 社区首页帖子流

- 发心得页

- 点赞

- 发布后帖子进入社区流

说明：

- 当前社区已经不是单纯展示页，而是具备最小可用闭环的社区页

### 4.8 我的页
已实现：

## 5. 当前已具备的 MVP 结论
当前项目已经满足简单 MVP 的阶段性验收条件。

原因：

- 核心做菜主链路已跑通
- 搜索闭环已跑通
- 导入闭环已跑通
- 社区闭环已跑通
- “我的”页已开始承接用户资产
- 页面结构已具备 Android / App 化的基本布局逻辑

当前状态可定义为：

- `可演示`
- `可验收`
- `可继续扩展`

但仍不是最终版产品，后续还可继续补：

- 收藏持久化
- 登录态持久化
- 社区评论
- 更完整的导入真实接口
- 更完善的数据落库

---

## 6. 下一阶段建议
建议下一阶段优先按以下顺序继续做：

1. 收藏与点赞等用户资产的持久化
2. 登录态与最近搜索的本地持久化
3. 社区评论与帖子二级交互
4. 导入接口真实后端接入
5. 做菜中断恢复的进一步增强

---

## 7. 验收方式建议
建议验收时按下面顺序实际走一遍：

1. 首页 -> 菜谱详情 -> 跟做 -> 完成
2. 首页 -> 搜索 -> 搜索结果 -> 详情
3. 首页 -> 导入 -> 识别 -> 预览 -> 详情
4. 社区 -> 发心得 -> 社区流 -> 我的发布
5. 我的 -> 收藏 / 最近做过 / 导入历史

如果这 5 条链路都能顺利完成，就说明当前 MVP 验收基本通过。
