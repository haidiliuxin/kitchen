# 小白下厨核心交互流程

这份文档补充说明当前项目的核心交互链路。当前版本不只有“从已有菜谱开始跟做”的路径，也包含“从外部视频或文档导入并生成攻略”的路径。

## 流程总览

- 路径 A：用户从首页、搜索或筛选结果中选择已有菜谱，进入详情，再进入跟做模式。
- 路径 B：用户从导入页粘贴视频链接或文档链接，系统解析内容并生成结构化攻略，再进入详情和跟做模式。
- 两条路径会在“详情页 / 跟做模式”汇合，后续都会进入步骤执行、AI 问答、完成记录和推荐闭环。

## 路径 A：已有菜谱到完成跟做

1. 用户进入首页，浏览推荐菜谱。
2. 用户通过搜索、难度和时长筛选缩小范围。
3. 用户进入详情页查看材料、步骤、提示和替代方案。
4. 用户进入跟做模式，切换为一步一屏。
5. 用户在每一步查看动作说明、补救建议和演示内容。
6. 用户通过按钮或语音切换到下一步，也可以发起 AI 提问。
7. 所有步骤完成后，系统写入做菜历史。
8. 系统基于历史记录给出下一道菜推荐。

## 路径 B：导入视频 / 文档生成攻略

1. 用户进入导入页，选择视频链接或文档链接。
2. 用户粘贴链接并提交导入请求。
3. 后端解析链接内容：
   - 文档链接优先提取正文、标题和结构化段落。
   - 视频链接优先提取标题、字幕和页面元信息。
   - 当平台字幕不稳定时，可调用本地 Whisper 服务做转写。
4. 后端结合解析结果生成结构化菜谱与分步攻略。
5. 前端展示生成结果，用户可以先预览再进入详情页。
6. 生成后的攻略进入与已有菜谱相同的详情页和跟做流程。
7. 后续步骤同样支持语音控制、AI 问答、完成记录和推荐。

## 交互价值

- 降低新手门槛：用户既可以直接做已有菜谱，也可以把收藏的视频或文章转换成可执行攻略。
- 统一主流程：导入生成的内容不会形成割裂页面，而是汇入同一套详情与跟做体验。
- 形成闭环：不论入口来自已有菜谱还是外部内容，最终都会沉淀到历史记录与推荐链路。
